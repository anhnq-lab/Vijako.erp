Đặc Tả Kỹ Thuật Hệ Thống Vijako ERP

Tài liệu định nghĩa kiến trúc kỹ thuật, mô hình dữ liệu và các nguyên tắc phát triển cốt lõi cho dự án.

1. Nguyên Tắc Cốt Lõi (Core Principles)

  • Dữ liệu tập trung (Centralized): Loại bỏ sự phân mảnh, mọi dữ liệu dự án chỉ nằm trên một nguồn duy nhất (Supabase).
  • Thời gian thực (Real-time): Ưu tiên cập nhật tức thì cho dashboard tài chính và thông báo hiện trường.
  • Trải nghiệm người dùng (UX First): Giảm thiểu thao tác nhập liệu, tự động hóa tính toán tối đa cho người dùng cuối.
  • Bảo mật (Security): Áp dụng Row Level Security triệt để cho từng dự án.

2. Công Nghệ & Stack (Tech Stack)

(Cấu hình này là tiêu chuẩn cho phiên bản 2.0)

  • Frontend: React 18, TypeScript, Vite.
  • Styling: Tailwind CSS (hệ thống Design System riêng).
  • Backend: Supabase (PostgreSQL + Auth + Storage).
  • State Management: TanStack Query (cho server state), Context API (cho UI state).
  • Maps: Leaflet với lớp phủ vệ tinh Google Hybrid.

3. Đặc Tả Chức Năng (Functional Specs)

3.1. Phân hệ Thanh toán (Payment Claims)

  • Cấu trúc: Quản lý theo WBS (Work Breakdown Structure) và danh mục BOQ.
  • Logic tính toán: Tự động trừ Advance Repayment và Retention.
  • Quy trình: Draft → Submitted → Certified → Approved.

3.2. Phân hệ Nhân sự (HRM)

  • Định danh: Mỗi nhân sự gắn với một employee_code duy nhất.
  • Chấm công: Tích hợp GPS khi check-in tại công trường.

4. Quy Chuẩn Code (Coding Standards)

  • Naming Convention:
    • Biến/Hàm: camelCase (ví dụ: calculateTotal, fetchProject).
    • Component: PascalCase (ví dụ: ProjectCard.tsx).
  • Commit Message:
    • Format: type: description (ví dụ: feat: add payment calculation logic).
